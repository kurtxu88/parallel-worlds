# Launch Checklist

Use this checklist before any meaningful public push such as Product Hunt, Hacker News, X, Reddit, a blog post, or a release announcement.

## Before Launch

- verify `docker compose up --build` works from a clean setup
- verify `npm run build`, `npm run test`, and `python3 -m compileall apps/api workers/story-generator`
- confirm the README still matches the current product and screenshots
- confirm starter seeds still produce compelling results
- confirm the social card and site metadata match the current positioning
- draft the release note before posting publicly

## Launch-Day Assets

- one short plain-English description
- one short builder-focused description
- one screenshot or clip
- one reliable starter-seed link
- one direct link to the roadmap or contribution guide

## Launch-Day Actions

- publish the announcement with a concrete hook
- stay available for issue responses during the first few hours
- collect repeated confusion points in real time
- turn good questions into README or docs improvements the same day

## After Launch

- record where attention came from
- note which demo seed or angle performed best
- follow up on every external issue or showcase quickly
- summarize what was learned and turn it into the next shipped improvement
