# 路线图

Parallel Worlds 会以公开构建的方式持续推进，目标是成为一个小而完整、可运行、可扩展的开源 AI 互动叙事栈。

我们并不想一开始就把它做成一个庞大的框架，而是希望每次发布都能让它更容易运行、更容易扩展，也更值得被分享出去。

## 产品方向

当前重点围绕四个方向展开：

- 首次体验足够顺滑
- 生成出来的是“可游玩的世界”，而不是静态文本演示
- 前端、API、worker、存储边界清晰
- 贡献范围友好，外部开发者能快速看懂

## 当前阶段

接下来几个公开版本的优先事项：

- 用示例种子、更清晰的文档和更顺手的本地启动方式优化首次上手体验
- 增强生成失败与重试流程的可观测性
- 为 API 和 worker 核心闭环补更多测试
- 提供更好的演示素材，让别人能一眼看懂项目在做什么

## 下一阶段

当核心闭环稳定后，下一步重点是提升“可分享性”：

- 在分享页之上继续做公开浏览与发现层
- 已生成世界的导出与导入
- 更丰富的示例种子和标准演示场景
- 更好的世界标题、摘要与浏览信息

## 更后面的方向

更长期的工作会在不破坏开源可理解性的前提下继续扩展：

- 自定义生成器或提示词包的扩展点
- 面向不同题材的故事/世界模板
- 面向自托管和小团队部署的指南
- 更完善的 worker 监控与运维工具

## 当前不优先做的内容

公开版 v1 暂不优先推进：

- 托管认证默认接入
- 邀请机制
- 仅适用于私有 SaaS 的工作流

如果某个方案依赖这些能力，建议先开 issue 对齐设计和取舍。

## 当前最有价值的贡献

在当前阶段，最有帮助的贡献类型包括：

- 提升首次本地运行成功率的修复
- 围绕故事生成状态流转的测试
- 减少创建、等待、进入世界过程中困惑感的 UI 改进
- 带有具体示例和排障内容的文档补充

## 我们如何决定优先级

我们通常会优先选择能提升以下维度的改动：

- 可演示性：别人能否快速理解并展示这个项目
- 可复现性：贡献者能否稳定地在本地跑通同样的流程
- 可扩展性：是否让更多构建行为能建立在核心闭环之上
- 清晰度：是否让这个公开代码库更容易学习，而不是更难
