# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for security-sensitive problems.

Report issues privately with:

- a short description of the issue
- affected files or services
- steps to reproduce, if available
- impact assessment, if known

If the report involves leaked credentials, rotate them first when possible and mention that in the report.

## What Counts as Security-Sensitive

- exposed secrets or tokens
- unsafe default credentials in docs or code
- auth/session bypasses
- remote code execution
- injection vulnerabilities
- data leaks between guest identities

## Response Goals

We aim to acknowledge reports quickly and follow up with a fix or mitigation plan as soon as practical.
